# Composer 工作流示例

## 功能开发工作流

### 添加新功能

```
步骤 1：规划
"分析项目结构，规划 {feature_name} 功能的实现方案"

步骤 2：创建数据模型
"创建 {model_name} 数据模型，包含以下字段：{fields}"

步骤 3：创建 API
"创建 {model_name} 的 CRUD API 端点"

步骤 4：创建前端组件
"创建 {model_name} 的管理界面，包含列表、创建、编辑功能"

步骤 5：添加测试
"为 {model_name} 的 API 和组件添加测试"
```

---

## 重构工作流

### Class 组件转 Hooks

```
步骤 1：分析
"分析 src/components 下的所有 Class 组件"

步骤 2：转换
"将 {component_name} 从 Class 组件转换为 Hooks"

步骤 3：更新生命周期
"将 componentDidMount 转换为 useEffect"

步骤 4：更新状态管理
"将 this.state 转换为 useState"

步骤 5：测试
"运行测试确保功能正常"
```

---

## 迁移工作流

### JavaScript 迁移到 TypeScript

```
步骤 1：配置
"为项目添加 TypeScript 配置"

步骤 2：迁移工具文件
"将 src/utils/*.js 迁移到 TypeScript"

步骤 3：迁移组件
"将 src/components/*.jsx 迁移到 TypeScript"

步骤 4：添加类型定义
"为 API 响应添加类型定义"

步骤 5：修复类型错误
"修复所有 TypeScript 编译错误"
```

## 国际化工作流

### 添加多语言支持

```
步骤 1：安装依赖
"安装 next-intl 并创建基础配置"

步骤 2：创建语言文件
"创建 messages/en.json 和 messages/zh.json"

步骤 3：配置中间件
"创建 i18n 中间件处理语言切换"

步骤 4：更新组件
"将硬编码文本替换为国际化调用"

步骤 5：添加切换组件
"创建语言切换组件"
```

## 测试覆盖工作流

### 添加测试覆盖

```
步骤 1：分析
"分析当前测试覆盖率"

步骤 2：单元测试
"为 src/utils 添加单元测试"

步骤 3：集成测试
"为 API 端点添加集成测试"

步骤 4：E2E 测试
"为关键用户流程添加 E2E 测试"

步骤 5：CI 集成
"配置 CI 流程运行测试"
```
